kbx ユーザーズマニュアル

このページはアプリ内の「?」ボタンから開ける、オフラインでも読めるヘルプです。 kbx の保管庫は標準 KDBX4 形式で、kbx 専用ではなく KeePass 系のどのアプリでも開けます。 バックアップ・復元・「アプリを使わず平文で見たいとき」の手順もここにまとめています。

  1. 大原則
  2. バックアップ（エクスポート）
  3. 復元（インポート）
  4. 平文で見たいとき（他アプリで開く）
  5. TOTP（2要素認証コード）の登録
  6. Face ID / Touch ID 解錠
  7. Dropbox 同期（複数端末で使う場合）
  8. マスターパスワードを変更したい場合
  9. マスターパスワードを忘れた場合
  10. トラブルシューティング

1. 大原則

  1. マスターパスワードを忘れたら誰にも復元できません。紙に書いて金庫など安全な場所に保管しておいてください。
  2. この端末の保存領域は「作業コピー」です。真のマスターデータは自分でエクスポートした .kdbx ファイルです。端末紛失・OS のストレージ整理・ブラウザデータ削除で端末内データは消え得ます。
  3. .kdbx ファイル自体は暗号化されているので、iCloud Drive 等に置いても中身は読まれません （ただしパスワードが弱ければ総当たりされ得ます）。

2. バックアップ（エクスポート）

  1. 保管庫を解錠 → 右上「エクスポート」
  2. iPhone: 共有シートが開く →「"ファイル"に保存」→ iCloud Drive の専用フォルダ（例: iCloud Drive/kbx/）へ保存
  3. PC: kbx-YYYY-MM-DD.kdbx がダウンロードされます

運用の目安:

  • エントリを追加・変更したらエクスポートする（5 件たまると画面に警告が出ます）
  • ファイル名に日付が付くので、直近 2〜3 世代を残して古いものは削除する
  • エクスポート直後に一度開けるか確認する（別アプリで開くか、kbx の「バックアップから復元」でパスワードが通るかを見ると確実です）

3. 復元（インポート）

  • 新しい端末・ブラウザデータ消去後: 初回画面の「既存の .kdbx をインポート」
  • 保管庫がロック中でも: ロック画面の「バックアップ (.kdbx) から復元」 （端末内の保管庫が選んだファイルで置き換えられます。確認チェックあり）

いずれも、その .kdbx を作った時点のマスターパスワードが必要です。

3.1 他のパスワードマネージャーからの移行（CSVインポート）

解錠中の「設定 → CSVからインポート」で、以下のCSVエクスポートを既存の保管庫に追加できます （.kdbx インポートと違い、置き換えではありません）。

  • Google パスワードマネージャー（Chrome / Android）: Chrome の「設定 → パスワードマネージャー → 設定 → パスワードをエクスポート」で CSV を書き出す
  • Apple パスワード（iPhone / Mac）: Mac の「パスワード」アプリで「ファイル → すべてのパスワードをファイルに書き出す」（CSV）。 iPhone 単体では CSV 書き出しができないため、いったん Chrome に移行してから Google パスワードマネージャーで CSV を書き出す経路が使えます（TOTP は Apple 形式のみ移行されます）

重要: エクスポートした CSV はパスワードが平文で入っています。 インポートが終わったら CSV ファイルを必ず削除してください（ゴミ箱からも消す）。

4. 平文で見たいとき（他アプリで開く）

kbx はブラウザが必要ですが、.kdbx は以下のアプリで直接開けます。 「ブラウザが使えない」「一覧を印刷したい」「kbx が壊れた」ときの非常口です。

4.1 PC: KeePassXC（推奨・無料/OSS）

brew install --cask keepassxc   # macOS

GUI で .kdbx を開き、マスターパスワードを入力すれば全エントリを閲覧・編集・検索できます。 パスワード欄は目のアイコンで表示切替できます。

CLI でも可（keepassxc-cli が同梱されます）:

# エントリ一覧
keepassxc-cli ls kbx-2026-07-03.kdbx

# 特定エントリの内容を表示（-s でパスワードも平文表示）
keepassxc-cli show -s kbx-2026-07-03.kdbx "テスト銀行"

# パスワードだけをクリップボードへ（10秒後自動クリア）
keepassxc-cli clip kbx-2026-07-03.kdbx "テスト銀行" 10

4.2 iPhone: Strongbox / KeePassium（無料枠あり）

App Store から入手 →「ファイル」に保存した .kdbx を開くだけです。 閲覧・編集とも可能で、kbx で作った保管庫をそのまま日常利用することもできます。

4.3 全件を平文ファイルに書き出す（要注意）

「紙に印刷して金庫へ」のような用途です。KeePassXC CLI で:

# CSV へ全件エクスポート（タイトル/ユーザー名/パスワード/URL/メモ）
keepassxc-cli export -f csv kbx-2026-07-03.kdbx > vault-plain.csv

警告: 出力ファイルは完全な平文です。取り扱いは以下を厳守してください。

  • クラウド同期されるフォルダ（iCloud Drive/Google Drive/Dropbox 等）に置かない
  • 用が済んだら即削除する。macOS なら rm -P vault-plain.csv（通常の rm でも実用上は可。ゴミ箱経由にはしない）
  • 印刷する場合、プリンタの履歴・スプールにも残り得ることを意識する

5. TOTP（2要素認証コード）の登録

サービス側で 2FA を設定するときに表示される「セットアップキー」（Base32 文字列）か otpauth:// URI を、エントリ編集画面の「TOTP (2要素認証)」欄に貼り付けて保存します。 一覧に 6 桁コードと残り秒数が表示され、タップでコピーできます。この欄も KeePassXC / Strongbox と互換です （同じエントリを開けば同じコードが出ます）。

6. Face ID / Touch ID 解錠

解錠後の「設定」からマスターパスワードを再入力して有効化します（iOS 18+ / macOS Safari 18+ / Chrome）。 以降はロック画面の緑のボタンから顔・指紋で解錠できます。パスワード入力での解錠は常に併用可能です。

  • 仕組み: パスキーの PRF 出力で導出した鍵でマスターパスワードを暗号化して端末内に保存します。 生体認証を通らない限り復号できません
  • バックアップからの復元や保管庫の削除、マスターパスワードの変更を行うと自動的に無効化されます （再度「設定」から有効化してください）

7. Dropbox 同期（複数端末で使う場合）

サイト側で Dropbox 連携が設定されている場合のみ、設定に「Dropbox 同期」欄が出ます。

  1. 「Dropboxと連携する」をタップ → Dropbox のログイン・承認画面へ移動 → 承認後にアプリへ戻ります。 戻った直後は保管庫がロックされた状態になります（ページの再読み込みを伴うため）。 もう一度マスターパスワードで解錠してください
  2. 解錠後、設定を開くと「今すぐ同期」ボタンが出ます。タップすると1回で Dropbox上の内容をダウンロード → 手元の内容とマージ → アップロード、まで完結します。 裏で自動的に同期することは一切ありません（同期したいときだけ自分でボタンを押します）
  3. 2台目の端末でも同じ手順で連携し、それぞれで「今すぐ同期」を押せば両方の変更が失われずに統合されます （同じエントリを両方で同時に編集した場合を除き、基本的に消えることはありません）

アクセス範囲: 連携は Dropbox の「アプリ専用フォルダ」のみに限定されます。 Dropboxアカウント全体へはアクセスしません。設定の「連携を解除」でいつでも取り消せます。

8. マスターパスワードを変更したい場合

解錠後の「設定」→「マスターパスワードの変更」から、現在のパスワードと新しいパスワード （確認入力あり）を入力すれば変更できます（忘れていない前提。忘れた場合は次節を参照）。

  • 変更するとFace ID / Touch ID 解錠は無効になります （旧パスワードをラップして保存していたものが使えなくなるため）。必要ならもう一度「設定」から有効化し直してください
  • Dropbox と同期している場合、変更後の最初の「今すぐ同期」で 「異なるパスワードで暗号化されている」というエラーが出ることがあります。 その場合は Dropbox アプリ（Web/デスクトップ/モバイルいずれでもよい）でこのアプリ専用フォルダ内の vault.kdbx を削除してから、もう一度「今すぐ同期」を行ってください （削除後は単純にアップロードし直されます）

9. マスターパスワードを忘れた場合

復元手段は存在しません（設計上、ヒントもリセットもありません）。

  1. 過去のバックアップ .kdbx のパスワードなら思い出せる場合 → ロック画面「バックアップ (.kdbx) から復元」で古い状態に戻す
  2. どのパスワードも分からない場合 → データは事実上喪失します。 ロック画面の「マスターパスワードを忘れた場合」を開き、「削除」と入力して 端末内の保管庫を消去し、新規作成からやり直してください （ブラウザの「サイトデータを削除」でも同じことができますが、アプリ内のこの導線を使えば他サイトのデータに影響しません）

10. トラブルシューティング

症状 対処
インポートで「ファイルを読み込めませんでした」 拡張子が .kdbx か、KDBX4 形式か確認。KeePassXC で開けるか切り分ける
「マスターパスワードが違います」 そのファイルを作った時点のパスワードを使う（変更前のバックアップは古いパスワードのまま）
iPhone でアプリを開いたらデータが消えていた OS がストレージを整理した可能性。最新のバックアップ .kdbx から復元する
共有シートに「ファイルに保存」が出ない 一度キャンセルし、共有シートの「その他」から有効化する
kbx 自体が使えなくなった .kdbx を KeePassXC / Strongbox で開けばデータは無事（これが KDBX4 採用の理由）
