# Security Policy

kbx is a single-maintainer project. There is no version support matrix —
only the latest `main` branch is maintained and receives security fixes.

## Reporting a Vulnerability

Please report security vulnerabilities privately via
[GitHub Security Advisories](https://github.com/sutezo/kbx/security/advisories/new)
rather than public issues.

Include as much detail as possible (steps to reproduce, affected component,
potential impact) so the report can be triaged quickly.

There is no fixed SLA, but reports will be acknowledged as soon as possible
and a fix will be prioritized based on severity. You'll be credited in the
fix's release notes unless you prefer to remain anonymous.
