# 保管庫 (.kdbx) 運用ガイド

最終更新: 2026-07-03 / 対象: 利用者向け（設計は [ARCHITECTURE.md](ARCHITECTURE.md) を参照）

kbx の保管庫は標準 **KDBX4 形式**。kbx 専用ではなく、KeePass 系の
どのアプリでも開ける。このガイドはバックアップ・復元・
「アプリを使わず平文で見たいとき」の手順をまとめる。

## 1. 大原則

1. **マスターパスワードを忘れたら誰にも復元できない**。紙に書いて
   金庫など安全な場所に保管しておくこと
2. **この端末の IndexedDB は「作業コピー」**。真のマスターデータは
   自分でエクスポートした `.kdbx` ファイル。端末紛失・OS のストレージ整理・
   ブラウザデータ削除で端末内データは消え得る
3. `.kdbx` ファイル自体は暗号化されているので、iCloud Drive 等に
   置いても中身は読まれない（ただしパスワードが弱ければ総当たりされる）

## 2. バックアップ（エクスポート）

1. 保管庫を解錠 → 右上「**エクスポート**」
2. iPhone: 共有シートが開く →「"ファイル"に保存」→ iCloud Drive の
   専用フォルダ（例: `iCloud Drive/kbx/`）へ保存
3. PC: `kbx-YYYY-MM-DD.kdbx` がダウンロードされる

運用の目安:

- エントリを追加・変更したらエクスポートする（5 件たまると画面に警告が出る）
- ファイル名に日付が付くので、直近 2〜3 世代を残して古いものは削除
- **エクスポート直後に一度開けるか確認する**（後述の別アプリで開くか、
  kbx の「バックアップから復元」でパスワードが通るかを見る）と確実

## 3. 復元（インポート）

- 新しい端末・ブラウザデータ消去後: 初回画面の「**既存の .kdbx をインポート**」
- 保管庫がロック中でも: ロック画面の「**バックアップ (.kdbx) から復元**」
  （端末内の保管庫が選んだファイルで**置き換え**られる。確認チェックあり）

いずれも、その `.kdbx` を作った時点のマスターパスワードが必要。

## 4. 平文で見たいとき（他アプリで開く）

kbx はブラウザが必要だが、`.kdbx` は以下のアプリで直接開ける。
「ブラウザが使えない」「一覧を印刷したい」「kbx が壊れた」ときの非常口。

### 4.1 PC: KeePassXC（推奨・無料/OSS）

```sh
brew install --cask keepassxc   # macOS
```

GUI で `.kdbx` を開き、マスターパスワードを入力すれば全エントリを
閲覧・編集・検索できる。パスワード欄は目のアイコンで表示切替。

CLI でも可（`keepassxc-cli` が同梱される）:

```sh
# エントリ一覧
keepassxc-cli ls kbx-2026-07-03.kdbx

# 特定エントリの内容を表示（-s でパスワードも平文表示）
keepassxc-cli show -s kbx-2026-07-03.kdbx "テスト銀行"

# パスワードだけをクリップボードへ（10秒後自動クリア）
keepassxc-cli clip kbx-2026-07-03.kdbx "テスト銀行" 10
```

### 4.2 iPhone: Strongbox / KeePassium（無料枠あり）

App Store から入手 →「ファイル」に保存した `.kdbx` を開くだけ。
閲覧・編集とも可能。kbx で作った保管庫をそのまま日常利用することもできる。

### 4.3 全件を平文ファイルに書き出す（要注意）

「紙に印刷して金庫へ」のような用途。KeePassXC CLI で:

```sh
# CSV へ全件エクスポート（タイトル/ユーザー名/パスワード/URL/メモ）
keepassxc-cli export -f csv kbx-2026-07-03.kdbx > vault-plain.csv
```

**警告**: 出力ファイルは完全な平文。取り扱いは以下を厳守:

- クラウド同期されるフォルダ（iCloud Drive/Google Drive/Dropbox 等）に置かない
- 用が済んだら即削除する。macOS なら `rm -P vault-plain.csv`
  （通常の `rm` でも実用上は可。ゴミ箱経由にはしない）
- 印刷する場合、プリンタの履歴・スプールにも残り得ることを意識する

## 5. TOTP（2要素認証コード）の登録

サービス側で 2FA を設定するときに表示される「セットアップキー」
（Base32 文字列）か `otpauth://` URI を、エントリ編集画面の
「TOTP (2要素認証)」欄に貼り付けて保存する。一覧に 6 桁コードと残り秒数が
表示され、タップでコピーできる。この欄も KeePassXC / Strongbox と互換
（同じエントリを開けば同じコードが出る）。

## 6. Face ID / Touch ID 解錠

解錠後の「設定」からマスターパスワードを再入力して有効化する（iOS 18+ /
macOS Safari 18+ / Chrome）。以降はロック画面の緑のボタンから顔・指紋で
解錠できる。パスワード入力での解錠は常に併用可能。

- 仕組み: パスキーの PRF 出力で導出した鍵でマスターパスワードを暗号化して
  端末内に保存。生体認証を通らない限り復号できない
- バックアップからの復元や保管庫の削除を行うと自動的に無効化される
  （再度「設定」から有効化する）

## 7. マスターパスワードを忘れた場合

復元手段は存在しない（設計上、ヒントもリセットもない）。

1. 過去のバックアップ `.kdbx` のパスワードなら思い出せる場合 →
   ロック画面「バックアップ (.kdbx) から復元」で古い状態に戻す
2. どのパスワードも分からない場合 → データは事実上喪失。
   ロック画面の「**マスターパスワードを忘れた場合**」を開き、
   「削除」と入力して端末内の保管庫を消去し、新規作成からやり直す
   （ブラウザの「サイトデータを削除」でも同じことができるが、アプリ内の
   この導線を使えば他サイトのデータに影響しない）

## 8. トラブルシューティング

| 症状 | 対処 |
|---|---|
| インポートで「ファイルを読み込めませんでした」 | 拡張子が `.kdbx` か、KDBX4 形式か確認。KeePassXC で開けるか切り分ける |
| 「マスターパスワードが違います」 | そのファイルを**作った時点**のパスワードを使う（変更前のバックアップは古いパスワードのまま） |
| iPhone でアプリを開いたらデータが消えていた | OS がストレージを整理した可能性。最新のバックアップ `.kdbx` から復元する |
| 共有シートに「ファイルに保存」が出ない | 一度キャンセルし、共有シートの「その他」から有効化する |
| kbx 自体が使えなくなった | `.kdbx` を KeePassXC / Strongbox で開けばデータは無事（これが KDBX4 採用の理由） |
